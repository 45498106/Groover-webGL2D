Groover-WebGL2D
Performance sprites. webGL1 shader.

2D games are all about sprites. The more sprites you can render per frame the more CPU time you can give to the stuff that matters, more sprites (joking I mean more sophisticated game logic that needs more sprites).

This demo shows a batched sprite rendering shader, each sprite can be faded, scaled, rotated, and positioned independently. Sprites can come from regular and irregular sprite sheets

Each device will have different performance, and different browsers also have different performance. This demo uses an array to hold sprites, Chrome is slightly better than Firefox at handling arrays (Sept 2016) so the upper limit of sprites (at 60FPS) will be the array indexing of Javascript if you have a good GPU.
